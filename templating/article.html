Essay

Readable by machines, lovable by humans

March 18, 2026 · 6 min read

Three years ago, the question was whether search engines could read your JavaScript. Today the question is whether a language model can quote your page back to a user without hallucinating a paragraph you did not write. The answer is the same in both cases: put the words in the HTML.

The split between markup and behaviour

We compose pages from small light-DOM fragments — a header, a card, a footer — and inline them at build time. The browser sees one continuous document. A crawler sees one continuous document. Only the editor sees fragments.

JavaScript comes in later, via a single deferred script. It finds existing elements by their data-* attributes and adds listeners. It does not generate DOM, and it does not wrap anything in a shadow root.

If removing the script still leaves you with a readable page, you have the split right.

Where JSON-LD fits

Each page carries its own application/ld+json block in the <head>, generated from the same front-matter that produced the page. No runtime injection, no duplicate source of truth.

What we gave up

Shadow DOM encapsulation, mostly. It is a lovely feature for application UIs and a poor fit for content sites whose audience now includes assistants that never open a shadow root. For a marketing site or a journal, the tradeoff is easy.
