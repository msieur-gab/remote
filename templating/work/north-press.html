Case study · Feb 2026

North Press — an editorial platform in ten files

A static publishing system for a 40-year-old literary journal. Semantic pages, JSON-LD for every article, and a build script short enough to read on the train.

North Press is a quarterly journal that has been in print since 1984. When they asked us to rebuild their online archive, the brief was short: every back issue should be legible on any device without JavaScript, and every poem should have a URL you can cite in another century.

We composed the site from small light-DOM fragments — a header, a card, a poem block, a footer — and inlined them at build time. The browser sees one continuous document. A crawler sees one continuous document. Only the editor sees fragments.

The editorial pipeline

Writers submit in Markdown with a short YAML preamble. A build script walks the content tree, inlines the fragments, generates a JSON-LD block from the same front-matter that produced the page, and writes one HTML file per route. The whole build is 187 lines of JavaScript and finishes in about 400ms.

Nothing is assembled in the browser. The site is ten HTML files and a stylesheet when you view-source, which is also roughly how the editors think about it.

JSON-LD as a first draft

We ask editors to fill out the structured data before writing the lede. Headline, author, section, publication date, and two or three about keywords. That exercise clarifies the piece more reliably than any brief, and it gives us a machine-readable page before a single paragraph is written.

What we cut

The shadow DOM, for one. It is a lovely feature for application UIs and a poor fit for content sites whose audience now includes assistants that never open a shadow root. We also dropped the client-side router, the lazy-hydration framework, and the image-CDN SDK. What remained was an HTML file.

The footprint went from 1.8 MB of assets per page to 128 KB, almost all of it an Inter subset and a poster image. First contentful paint is the first byte off the wire.

Mobile, quietly

The layout is a single fluid column that breathes between 18 and 68 rem. There is no mobile template, no app shell, and no second stylesheet to keep in sync. The phone gets the same document the editor reads back to themselves at noon.

Two years in, the editorial team publishes four issues a year without calling us. The build is still 187 lines. The pages are still ten files.
